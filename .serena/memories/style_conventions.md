# Style and Conventions

## General
- The codebase is currently empty and will be built from scratch.
- Follow standard, modern conventions for the respective languages.

## Kotlin (Android & ML Integration)
- Adhere to the official Kotlin Coding Conventions.
- Use explicit typing for public APIs and complex logic.
- Keep ML initialization and inference logic on background threads (Coroutines/Dispatchers.IO).

## TypeScript/JavaScript (saizeriya.js integration)
- Use standard Prettier/ESLint configurations once the Node.js backend/scripts are initialized.
- Use async/await for network requests and CLI interactions.
